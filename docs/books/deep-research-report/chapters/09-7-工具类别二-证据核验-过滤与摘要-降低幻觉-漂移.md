# 7 工具类别二：证据核验、过滤与摘要（降低幻觉/漂移）

长视频推理的“工具化”趋势并不只体现在放大、拆帧等感知层操作上，更关键的一支演化是围绕可靠性而引入的证据核验、过滤与摘要机制：一方面，长视频往往需要先经由检索/时间定位把候选片段从海量时序上下文中“拉进窗口”，这一步天然带来误召回与噪声；另一方面，即便片段检索正确，模型在生成长链推理时也可能被语言先验牵引，产生与视觉事实不一致的“漂移式幻觉”。因此，许多 think-with-video 系统把“验证器（verifier）”“闭环一致性检查”“证据过滤与对齐”“片段级摘要”显式做成可调用工具或可分离执行器，以降低无关片段诱发的幻觉、抑制推理链漂移，并在一定程度上提升可解释性边界。

在 VideoExplorer 的框架化描述中，可靠性首先被嵌入到“推理链—检索链”解耦的系统结构里：作者提出的 temporal grounder 被设计为独立于主推理链的执行器，离线将视频切分为 clips 并存入多模态嵌入库，在线阶段再用文本或多模态查询去匹配候选片段；更重要的是，它把抽象的子问题查询转译为一组具体动作（construction、retrieval、verification、summarization），并将“经验证的片

段”返回给高层 planner，而不是让 planner 直接在大量候选片段上进行端到端自由生成式推理 [3]。这种解耦的价值在于把易错且成本高的证据定位过程收束为更可控的执行模块：当验证环节判定某段检索结果与当前子问题无关时，系统可以直接丢弃该片段，仅将通过验证的证据交给 planner，从而减少“把无关片段当成证据”所触发的幻觉与错误归因 [3]。需要强调的是，这里 verifier 的角色更接近“证据过滤器”而非“答案裁判”，它的边界在于只能就“相关/不相关”“是否满足子问题约束”做判别；如果 verifier 自身存在系统性误判，那么它同样可能带来两类相反风险：误保留噪声片段导致幻觉污染，或误过滤关键证据导致后续推理在信息缺失下被迫依赖语言先验而猜测。

闭环一致性检查常被用来把“工具调用是否真的提升了答案可靠性”转化为可操作的筛选准则，尤其体现在多轮工具调用轨迹数据的构造阶段。VideoTemp-o3 将 thinking-with-videos 明确表述为“localize-crop-answer”的交互过程，并在其多轮、单工具调用数据生成流程中引入 closed-loopconsistency verification：在用外部强模型提出候选时间段并裁剪后，训练样本只有在“最终答案与ground-truth 保持一致”时才被保留，从而形成多轮对话但单次工具调用的训练数据 [4]。对于长视频（文中明确阈值为 $> 3 \mathrm { { m i n } }$ ），该流程允许再进行一轮 refinement；若在额外一轮细化后通过一致性检查，则保留为 multi-turn、multi-tool-call 训练数据 [4]。这种“以闭环一致性为门槛”的工具，其作用边界同样清晰：它能有效剔除由错误裁剪、错误定位或工具调用失败引入的噪声轨迹，提升训练数据的“工具调用—答案正确”同向性；但由于一致性判据通常依赖既有标注或标准答案，它对开放式问题（无唯一答案、或答案可被多种证据支持）以及对证据解释多样性的覆盖会更弱，也可能引入选择偏差，使保留下来的轨迹更偏向“易于通过一致性验证”的样本分布。

证据过滤不仅发生在训练数据层，也直接影响推理期的工具选型与证据形态。VITAL 的工具消融结果显示，并非所有看似“补充信息”的视觉工具都会提升时序定位或减少幻觉：作者观察到在temporal grounding 任务上加入 clip caption 或 clip QA 工具不仅没有带来改进，反而会导致 mIoU显著下降；相较之下，video clipping 在经验上比其他工具更能减少 hallucination[2]。该对比提示两点：其一，“更多中间描述”并不必然等价于“更可靠的证据”，当 caption/QA 本身携带生成噪声或把注意力从关键视觉细节转移到语言化概括时，反而可能放大漂移；其二，过滤与裁剪类工具（把证据边界收窄到更高置信的时空区域）往往更像“降噪操作”，其增益来自减少模型在无关上下文中自由联想的机会。与此相呼应的是，VideoExplorer 的 verifier 过滤、VideoTemp-o3 的一致性筛选，都在不同阶段把“证据边界”收敛为更少、更相关的片段，实质上是在用工具把推理的自由度约束在较小的、可解释的证据集合上。

为了直接对抗推理链的“视觉思维漂移（visual thinking drift）”，也有工作把“证据对齐”提升为可学习的训练目标，而非仅作为工程规则。When Thinking Drifts 指出 CoT 在视频推理中可能产生“冗长但误导性的内心独白”，导致幻觉的视觉细节甚至覆盖原本正确的直觉，并将其概括为 visualthinking drift[10]。该文提出 Visual Evidence Reward（VER），核心是奖励那些“可被视觉证据验证的推理轨迹”，并将 visual evidence 定义为与问题相关、回答所必需的视觉细节（question-dependent）[10]。在证据生成上，作者使用外部强 MLLM（Qwen2.5-VL-72B）在给定视频与问题的条件下生成答案与支持该答案的视觉观察列表；但策略模型 Video-VER 在测试阶段不依赖外部证据生成器独立推理 [10]。这种“训练期工具（teacher）/推理期去工具”的设计边界较为明确：它把证据抽取能力外包给强模型以获得监督信号，同时避免推理期把外部生成器引入为新的不确定性来源；但其有效性仍受制于证据生成质量与“可验证性”定义是否覆盖任务所需的全部关键细节，一旦证据列表遗漏或表述含混，奖励信号可能会把模型引向过度简化的证据模板。

摘要（summarization）在 think-with-video 体系中常作为“证据压缩”工具出现，其目标不是生成最终答案，而是把通过检索/验证的片段转写为更短、更易于多轮累积的中间表示，从而在窗口长度受限时维持跨回合的证据可用性。VideoExplorer 明确把 summarization 作为 temporal grounder可执行的动作之一，并强调其返回的是“validated segments”供 planner 使用 [3]。从可靠性角度看，

片段级摘要的收益来自两方面：一是将模型的关注点从未过滤的原始帧序列转移到“已通过验证的片段内容”，降低无关细节干扰；二是在多步推理中提供可追溯的证据条目，使后续步骤更容易围绕同一组证据保持一致。然而摘要也存在典型边界：摘要是对原证据的二次生成，可能引入再一次的信息损失与措辞偏置；因此，在强调可解释性的系统里，摘要更适合被视为“缓存与路由用的压缩表征”，而不是替代原始证据的最终依据。

最后，证据核验与对齐并不只依赖视频领域专用模块，一些通用的“生成—验证—过滤”工具链也可迁移到视频推理的文本输出层。THaMES 在面向幻觉缓解与评测的端到端工具链中，展示了多种可工具化的质量控制手段：例如通过 RegEx 过滤掉包含歧义关键词、难以自包含的问题以降低评测集噪声，并引入 chain-of-verification prompting 来构造聚焦 ANSWER_ENTITY 与 QUES-TION_ENTITY 的核验问题 [18]；其评测指标还给出了基于“支持性陈述占比”的忠实度定义，如Answer Faithfulness $A _ { F } = | V | / | S |$ （ $| V |$ 为被支持的陈述数， $| S |$ 为总陈述数）[18]。对于 think-with-video 系统而言，这类机制可以被视为“输出侧证据对齐器”：当模型完成片段检索、裁剪、摘要后，仍需在最终文本中把主张与证据陈述对齐，并通过核验式提问或陈述级评分来检测是否出现了超出证据的扩展。与数据/评测层的可靠性设计相配合，GLIMPSE 也通过双向问题对（bidirectional pairs）来检验一致性，只有两问均答对才计为正确；在该设置下，作者报告 GPT-4o 若只看正向问题准确率为 $5 2 . 3 0 \%$ ，但按问题对计分仅 $2 8 . 8 0 \%$ ，揭示了单向评测可能高估模型稳定性的问题 [12]。这类一致性评测工具并不直接减少幻觉，但它为“是否需要引入 verifier、过滤器、摘要器”等工具提供了更苛刻、更能暴露漂移的诊断信号，从而反向驱动工具链的设计与取舍。
