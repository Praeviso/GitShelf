# 10 工具调用策略与训练对齐：从格式约束到轨迹级优化

在 think-with-video 框架中，工具调用并不是“把更多视觉算子挂到模型外面”就自然稳定发生的能力，而是一个需要被显式学习、被约束输出接口、并在多轮交互轨迹上被对齐的决策过程。许多工作把“推理（thought）—工具动作（tool call）—环境观测（tool response）—再推理”的循环视为一个序列决策问题：智能体在生成中间思考的同时，必须决定何时去视频环境中寻找新证据、调用哪一个工具、以及是否已经获得足够证据可以结束交互并给出最终答案 [11, 3]。该类过程一旦进入强化学习或偏好优化阶段，训练信号会同时作用于语言生成质量、证据获取质量与工具调用成本，若不设计额外约束，容易出现“工具乱用、过度调用、或用不相关证据拼出看似合理答案”的不稳定行为 [11, 4]。

首先，输出格式约束往往是工具调用得以规模化训练的前置条件，因为工具调用必须落到一个可解析、可校验的动作接口上，否则训练阶段难以为“动作是否正确执行、轨迹是否可复现”提供明确监督。VITAL 为了“稳定思考与工具调用”，明确要求采样轨迹以 <think>...</think><tool_call>{"name": 1…", "arguments": {…}}</tool_call>...<answer>...</answer> 的结构化形式输出，并配合严格的基于规则的格式奖励 $R _ { \mathrm { f o r m a t } } ( \tau )$ 进行约束；该做法将工具调用从自然语言中剥离成可执行的JSON 参数，使得后续的奖励计算与轨迹过滤可以以确定性方式进行 [2]。在临床长视频的 Med-Scope 中，结构化接口被进一步扩展为在每轮轨迹中显式区分 <think>（文本理据）、<tool_call>（视觉理据/动作）、<tool_response>（环境观测）与 <answer>（终止预测），并把“是否遵守接口”直接作为复合奖励的一部分，用于把工具调用行为纳入与正确性同等地位的优化目标 [6]。这类“接口—解析—奖励”闭环也能反向影响数据生产：当数据合成和训练都以结构化格式为中心时，轨迹级过滤、纠错与再标注将显著更容易自动化 [6, 4]。

仅靠格式约束仍不足以让模型学会“每一步该不该调用工具、调用后是否真的拿到了证据”。因此，逐步操作正确性奖励成为许多 think-with-video 强化学习算法的核心：它把轨迹中的工具动作当作可评估的中间目标，而不是只看最终答案对错。Video-CoM 把交互式视频推理建模为“Chainof Manipulations”，每步从三种原子操作中选择其一：按段重访的 find-segment、冻结关键帧的find-frame、以及对局部区域裁剪放大的 spatial-zoom[5]。为了避免“只要最终答对就不区分中间步骤好坏”的信号稀疏问题，Video-CoM 在 GRPO 基础上提出 RA-GRPO，通过逐步推理奖励$R _ { \mathrm { r e a s o n } }$ 显式评估每个 manipulation 的正确性，并以 $\begin{array} { r } { R _ { \mathrm { r e a s o n } } = \frac { 1 } { N } \sum _ { i } C _ { i } } \end{array}$ 的形式将所有步骤的正确性平均汇总，使策略梯度能直接朝“操作序列更对齐证据”的方向更新 [5]。MedScope 的复合奖励也在同一思想下引入证据项 $R _ { \mathrm { e v i d e n c e } }$ ：它不只问“答案是否正确”，而是根据最后一次工具调用是否与标注证据时间对齐来计分；对 crop_video 用时间重叠/IoU 评估，对 get_frame 用时间戳对齐评估，并对裁剪类工具加入 IoU 递进式 bonus schedule，从而把“粗到细定位—取证—核验”的行为链条转化为可学习的强化学习信号 [6]。

当奖励开始直接绑定 temporal IoU、时间戳对齐等指标时，另一个问题随之出现：模型可能通过“投机性定位”在指标上获得高分，而并未真正形成可解释、可复用的证据定位策略，即强化学习中的 reward hacking。VideoTemp-o3 明确指出“仅用 IoU 作为奖励”会导致模型“任意定位来hack reward”，因此提出 penalty-aware IoU reward：在 IoU 奖励之外对低质量的定位行为施加显式惩罚，以抑制胡乱猜测式的定位并鼓励更可靠的多轮细化 [4]。这一点提示：工具学习中的奖励设计往往需要同时包含“对齐奖励”与“行为约束项”，前者推动模型接近目标，后者避免模型在指标空间钻漏洞。类似的“成本/惩罚”也体现在推理效率维度：Video-Thinker 的 RL 训练除了答案正确性外，还会对过度工具调用施加惩罚以平衡性能与效率，避免智能体在简单问题上“工具过调用”造成不必要开销 [11]。

多工具、多任务的 think-with-video 训练还面临一个更宏观的优化难题：不同任务和不同样本的学习难度差异会在多任务强化学习中放大，导致训练过程被“容易提升的子任务”主导，从而让工具策略在真正困难的长视频定位任务上学不到关键能力。VITAL 在多任务 RL 中观察到 task-wise难度不均衡：短视频或简单任务（例如多选题）奖励上升很快，但长视频 temporal grounding 的 IoU奖励增长更慢；同时还存在 sample-wise 不均衡，即随着 RL 训练推进，易样本比例升高而难样本比例下降 [2]。为此其提出 Difficulty-aware GRPO（DGRPO）来处理难度不均衡，并在 GRPO 目标中保留与参考策略的 KL 约束项，以稳定策略更新。其优化目标写为

$$
J _ {\mathrm {G R P O}} (\theta) = \mathbb {E} _ {q \sim p (Q), \tau_ {k} \sim \pi_ {\theta} (\cdot | q)} \left[ \frac {1}{G} \sum_ {k = 1} ^ {G} \frac {\pi_ {\theta} (\tau_ {k} \mid q)}{\pi_ {\theta_ {\mathrm {r e f}}} (\tau_ {k} \mid q)} A _ {k} - \beta D _ {\mathrm {K L}} (\pi_ {\theta} \| \pi_ {\theta_ {\mathrm {r e f}}}) \right], \tag {2}
$$

并通过难度感知的奖励平衡，使多任务训练不至于被“容易的奖励通道”吞噬，从而在需要工具定位

与证据裁剪的任务上保持学习压力 [2]。从可复现的训练细节看，VITAL 在 SFT 与 RL 阶段采用不同学习率（SFT 为 $1 \times 1 0 ^ { - 5 }$ ，RL 为 $1 \times 1 0 ^ { - 6 }$ ）、不同 batch size（SFT 256，RL 64），并在 DGRPO中使用 8 个 rollouts；这些工程化设置反映出工具学习通常对训练稳定性更敏感，需要在采样数与KL 正则之间做更谨慎的折中 [2]。

除强化学习外，结构化模仿学习与轨迹级偏好优化正在成为“把工具使用变成稳定技能”的另一条主线，因为工具调用本质上是多轮轨迹的结构问题：一条轨迹是否好，不仅取决于答案是否正确，也取决于子问题分解、证据检索、验证与结束时机是否合理。VideoExplorer 在训练上首先进行结构化模仿的 SFT，强调学习复现“专家推理痕迹（reasoning traces）”而非仅拟合最终答案；随后引入 trajectory-based direct preference optimization（TDPO），把 DPO 从单轮输出扩展到多轮、多步视频推理场景，对“整条推理轨迹”进行偏好学习，使训练目标能够同时覆盖中间步骤质量与最终结果 [3]。与之相呼应，VideoTemp-o3 在多轮对话式工具调用数据上采用统一 masking：只对最后两轮输出施加损失，前序回合与输入不参与梯度，从而降低早期回合定位噪声对学习的干扰；这可以理解为在监督信号有限时，对轨迹中“决定性步骤”进行加权的对齐策略 [4]。

上述对齐方法的效果高度依赖“单工具调用/多工具调用”训练数据如何构建，以及质量控制是否能覆盖工具动作层面的正确性。VideoTemp-o3 的数据流水线体现了这种数据—对齐共设计：其多轮工具调用样本由强模型先预测候选时间片段并裁剪，再强制模型“仅基于该片段作答”；随后进行闭环一致性验证（closed-loop consistency verification），只有当最终答案与真值一致时才保留样本，形成 multi-turn、single-tool-call 数据；对长视频（ $\mathrm { > } 3 \mathrm { m i n }$ ）再允许额外一轮 refinement，通过“再定位—再裁剪”保留通过样本为 multi-tool-call 数据 [4]。Video-CoM 的数据构建同样强调端到端的轨迹可控性：每个视频预先等长分段，并把 segment 与 frame index 直接叠加到帧上，使 find-segment、find-frame 等动作具有可标注、可比对的离散目标；其 Video-CoM-Instruct-18K 从约 9K 视频出发，用 Gemini-2.5-Pro 为每个视频生成 3–5 个 QA（总计 25K），经验证后保留约 $7 5 \%$ 得到 18K高质量样本，并明确划分 15K 用于 SFT、3K 用于 GRPO/RA-GRPO[5]。VideoExplorer 的数据更强调覆盖复杂推理：通过 difficulty-adaptive sampling 先均匀采样任务、生成轨迹，再根据首轮准确率对困难样例重采样，并“仅保留正确答案轨迹”以确保有效性；其最终整理出 11.1k plannertrajectories 与 10.8k grounding trajectories，为分模块训练（规划器与时间定位器）提供了可分离的监督材料 [3]。

质量控制除了“答案一致性/正确性过滤”，在工具学习里还必须防止模型利用数据捷径学会绕开工具或伪造证据。MedScope 的 ClinVideoSuite 合成流程中，先构造带时间戳的密集字幕作为局部证据并用低采样摘要保留全局上下文，再通过跨模型投票的级联过滤去除知识驱动问题、摘要模型捷径与内部不一致样本，最后进行多模态确认：直接从视频片段 ${ \mathrm { C l i p } } ( V , w )$ 回答问题，以消除字幕幻觉并保证问题确实依赖视频证据；在此基础上再通过真实环境交互合成 visual-CoT 轨迹，得到既包含 tool-augmented 也包含 tool-free 的轨迹分布，用于训练“何时需要工具”的决策边界 [6]。这类数据验证思路与 VideoExplorer 的“检索—验证—总结”执行链相呼应：其时间定位代理在推理时会用 VLM 作为 verifier 丢弃无关片段，把通过验证的片段回传给规划器，从系统层面把“错误证据”阻断在进入主推理链之前 [3]。

工具调用策略的学习还可以被理解为一种“路由/预算分配”问题：不是所有查询都应调用所有工具，也不是每次调用都需要最昂贵的模态或最高密度的观测。ModaRoute 将这一点在检索任务中做了量化：它用 LLM 路由器根据查询意图在 ASR、OCR 与视觉索引之间做选择性检索，平均每次查询只用 1.78 个模态（而穷举是 3.0），在保持 R $\mathrm { \ t e c a l l @ 5 = 6 0 . 9 \% }$ 的同时把计算成本降低约 $4 1 \%$ ，并指出路由失败主要集中在 OCR 需求识别（单模态路由中视觉 $9 9 . 4 \%$ 准确、ASR $7 1 . 7 \%$ 、OCR$4 2 . 9 \%$ ）以及字幕导致的 ASR/OCR 混淆 [7]。这类结果对视频智能体同样具有启示：工具调用对齐不仅是“动作能否对上 GT 区间”，也包含“是否能识别信息源在哪、是否值得花成本去查”的策略

学习，且误差常来自模态间语义重叠与保守偏置 [7]。

最后，许多工作开始把“证据对齐”从系统工程上升为训练目标，以缓解多轮推理中常见的幻觉与推理漂移问题。When Thinking Drifts 的分析指出，视频推理中的 CoT 往往因为缺乏对 CoTtoken 的显式监督而偏离视觉证据，生成冗长但误导性的内心独白并覆盖原本正确的直觉；其提出Visual Evidence Reward（VER），用强化学习显式奖励“可验证地 grounded 在视觉证据上”的推理轨迹，并用强外部 MLLM（Qwen2.5-VL-72B）离线生成问题相关视觉证据作为训练依据，而测试时策略模型独立推理不依赖外部证据生成器，从而把“工具/教师模型”限定在训练期以降低推理期依赖 [10]。从更广义的工具观出发，Video Language Planning 甚至把视频（尤其是想象 rollout）作为语言计划与低层动作之间的中间表征，指出这种视频化表征天然支持放大、抽帧、按时间查询场景属性等视觉工具操作，并可将这些操作应用到“想象视频”上做可行性审查；这为 think-with-video的训练对齐提供了另一条路径：工具不仅用于回答问题，也可用于规划阶段的自我检查与反事实比较，但其有效性同样受限于生成/预测视频在长时域上的误差累积 [29]。

综上，think-with-video 中工具调用的可学习性与稳定性，依赖一套从“接口格式 奖励分解反作弊惩罚 难度平衡 轨迹级对齐 数据构建与质量控制”的系统性设计。格式奖励把动作空间变成可解析对象并为自动化验证铺路 [2, 6]；逐步奖励与证据奖励让工具动作本身获得密集、可比较的训练信号 [5, 6]；惩罚项与成本项分别应对 reward hacking 与过度调用等典型失效模式 [4, 11]；难度感知的多任务 RL 试图把训练压力重新分配给真正困难的长视频定位/取证任务 [2]；而结构化模仿与轨迹级偏好优化则把“整条交互轨迹”作为对齐对象，使工具调用从局部动作正确扩展到全局策略合理 [3]。在这一框架下，“除 zooming、拆帧以外是否有更多工具”的问题最终会落到训练对齐：工具越多，越需要可扩展的轨迹标注、自动验证与可解释奖励，才能避免工具箱扩大反而带来不稳定与幻觉累积。
