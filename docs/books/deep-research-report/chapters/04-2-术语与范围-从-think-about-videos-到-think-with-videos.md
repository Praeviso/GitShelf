# 2 术语与范围：从“think about videos”到“think with videos”

本文讨论的“think with video / thinking-with-videos”并非泛指“模型能回答视频问题”，而是强调推理过程与视觉证据获取过程被耦合为可反复交互的闭环：模型在推理中可以回看、聚焦、验证并据此更新后续推理步骤，而不是将视频一次性编码为静态上下文后，后续推理完全在文本空间展开。Video-CoM 将后者概括为多数方法仍在“think about videos”的被动范式：视频被编码后，

推理在文本中展开，形成“semantic bottleneck”，使模型无法“rewatch, refocus, or verify evidence”，在需要细粒度时空理解的任务上更易产生浅层推理与幻觉；相对地，“think with videos”把视频视作动态的“cognitive workspace”，通过迭代视觉动作来收集与精炼证据。[5]

在这一范式中，“tool（工具）”指一种可被显式调用的、对原始视频或其派生表示执行的操作模块，它在推理中充当“按需感知接口”，将“看哪里/看多久/看多细”从端到端编码中外部化为可控动作。Video-Thinker 将工具增强的视频推理形式化为序列决策过程：智能体在“生成中间思考（intermediate thoughts）”与“从工具集合中选择动作”之间交替，以获得新的视觉观测，并通过强化学习学会何时以及如何调用这些视频工具。[11] 因此，这里所说的工具并不限于传统意义上的视觉算子（如检测、分割），也包括更“编辑式/交互式”的视频操作，例如时间跳转（seek）、片段截取（trim/extract sub-clip）、区域裁剪放大（zoom/crop）等轻量操作。它们的共同特点是：一次调用会返回新的、任务相关的视觉上下文，从而改变后续推理的输入条件。[11]

“tool call（工具调用）”指一次结构化的工具请求，以及其执行结果进入推理上下文的过程。工具调用的结构化表达通常是实现稳定闭环的关键工程约束：例如 VITAL 为稳定“thinking 与 toolcalling”的交替，要求模型以特定的标签与字段格式输出（例如用 <think>、<tool call>、<answer>等区块，并在 tool call 中以 JSON 样式给出 name 与 arguments），并引入基于规则的格式奖励来强化该约束。[2] 这种严格格式把“推理文本”与“可执行动作”解耦，便于复现工具序列、做消融分析与进行可解释的证据链追踪。

本文进一步采用“manipulation（操作/操控）”来指代可被模型选择的原子级视频动作集合，强调其可组合性与轨迹结构。Video-CoM 将交互式推理建模为“Chain of Manipulations（CoM）”，每一步从若干原子操作中选择其一，并把多轮过程形式化为轨迹 $\tau = \{ ( T _ { 1 } , V _ { 1 } , m _ { 1 } ) , \ldots , ( T _ { N } , V _ { N } , m _ { N } ) \}$ ，其中 $T _ { i }$ 为当轮中间推理文本， $m _ { i }$ 为当轮操作， $V _ { i }$ 为该操作得到并反馈的新视觉上下文。[5] 在其实现中，原子操作包括 find-segment（按段索引取回短时间窗以重访/回看关键片段）、find-frame（定位单帧以获得高分辨率静态证据）与 spatial-zoom（对局部区域裁剪放大以检查细节）。[5] 这种“操作”概念强调：工具使用并不等价于“多抽几帧”，而是以可执行动作将注意力、时间窗口与空间分辨率在推理过程中显式重分配。

“temporal grounding（时间定位/时间区间定位）”在本文中特指：把自然语言问题或子问题映射为视频时间轴上的一个或多个区间（或段索引），并将其作为后续感知与回答的证据载体。Video-Explorer 将“thinking with video”描述为面向“on-demand temporal context”的动态推理过程：规划器将复杂问题分解为子问题，时间定位模块自适应地局部化相关时间跨度，随后感知模块在这些时间跨度内以可伸缩粒度进行理解，从而避免对全片段进行蛮力式下采样或静态预处理。[3] 与此相伴的“on-demand perception（按需感知）”指模型并不固定以均匀采样覆盖整段视频，而是由任务驱动地产生可伸缩的感知查询；在 VideoExplorer 的叙述中，不同任务会触发不同采样粒度：细粒度问题在定位到的时间窗内进行更密集采样，而全局性问题可覆盖更宽时间窗并采用更粗采样，以在上下文窗口受限下实现“scalability–fidelity”的折中。[3]

为更清楚地区分“工具调用”与“被动编码”，本文用“交互式感知—推理闭环”指代一类系统范式：推理链条中至少包含一次显式的视频定位/裁剪/采样调度动作，并在得到新观测后继续推理与再决策。VideoTemp-o3 将 thinking-with-videos 直接任务化为“localize-crop-answer”过程，并强调在长视频中可通过迭代调用工具（例如视频裁剪与密集采样）来弥补均匀采样常遗漏关键证据的缺陷；其框架支持多轮时间定位细化（multi-turn grounding refinement），直到收集到足够证据再作答。[4] 这一表述对应本文的关键边界：我们关注“推理过程中能反复定位与取证”的机制，而不是仅在输入端做一次性预处理（如离线抽帧、全局特征池化）后再进行单轮文本推理。

除“获取证据”的工具外，本文也将“证据核验/对齐”视为 think-with-video 工具链中同等重要的组成部分，因为长视频推理常出现“推理漂移”与视觉幻觉。VideoExplorer 在时间定位模块中

引入验证环节：用 VLM 作为 verifier 评估检索片段是否相关，丢弃无关结果并只把通过验证的片段返回给高层规划器，从而减轻无关上下文引发的幻觉风险。[3] When Thinking Drifts 则从链式思维在视频推理中可能“生成冗长但误导性的内心独白、产生幻觉细节并覆盖正确直觉”出发，将该现象命名为 visual thinking drift，并提出 Visual Evidence Reward（VER）用强化学习显式奖励“可被视觉证据验证支撑”的推理轨迹；其关键设定之一是将“visual evidence”定义为问题相关（question-dependent），并在训练阶段利用强 MLLM 生成支撑性视觉观察列表，使策略模型的推理更贴近可验证证据。[10] 在本文术语体系中，这类机制被归入“evidential grounding（证据化落地）”：不仅要回答问题，还要让推理链中的关键断言能对应到可检查的视觉事实，从而使工具调用序列具备可审计性。

研究范围方面，本文聚焦于“长视频推理（long video reasoning）/长时程视频理解”语境下的工具使用：一是面向长时长带来的计算预算与上下文窗口限制，工具用于实现时间定位、裁剪、密集采样与证据重访；二是面向证据稀疏与误检索风险，工具用于验证、过滤与证据对齐；三是面向可解释性与可复现分析，工具调用需要以结构化接口与轨迹形式呈现。[3, 4, 11] 与此相对，GLIMPSE对“thinking with video”的评测定义强调：问题必须要求观看整个视频并在全视频上下文上推理，不能通过扫描少数关键帧或依赖文本捷径作答；这类基准为诊断“是否真正使用全视频证据”提供了参照——如果某种所谓“工具使用”最终仍退化为“glimpse 少数帧即可答”，则难以支撑其“thinkwith video”的主张。[12] 因而，本文不把“视频到文本的一次性描述/摘要然后用文本推理”视为think-with-video 的充分条件；同样也不把“仅增加输入帧数”当作工具化推理的替代。

最后，本文讨论的“工具”限定为能被系统显式调用并改变后续感知输入的模块或步骤（例如时间定位、裁剪、密集采样、片段检索与验证、以及面向证据的奖励/核验），而不扩展到所有模型内部的隐式能力（例如仅靠提示词引导注意力但不产生可执行动作的做法）。这种限定使不同工作可在统一口径下比较其工具集合、调用策略与可解释的证据链条，并为后续章节梳理“除 zooming、拆帧之外，视频推理中还存在哪些可复用工具”提供清晰边界。[2, 5, 11]
