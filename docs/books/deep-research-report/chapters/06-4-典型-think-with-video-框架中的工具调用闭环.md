# 4 典型 think-with-video 框架中的工具调用闭环

在“think-with-video”范式中，工具并非附着在模型外部的可选加速项，而是把“看视频”这一感知过程重新组织为可被语言推理显式调度的动作序列：模型（或多智能体系统）先提出下一步的信息需求，再通过工具在长视频中定位可疑证据段，随后以合适的时间/空间粒度重新感知并产出可核验的观察，必要时再执行验证、摘要或二次检索，最终在证据充分时给出答案。这种“规划—定位/检索—感知—验证/摘要—作答”的闭环之所以在长视频场景里变得关键，一方面来自上下文窗口和计算预算对“全量编码 $^ +$ 一次性推理”的限制，另一方面也来自视频推理中幻觉与“推理漂移”风险：若缺少证据回看与核验机制，模型容易生成冗长但缺乏证据支撑的推理文本，甚至覆盖原本正确的直觉判断，从而偏离真实视觉证据 [10]。

以 VideoExplorer 为代表的一类系统把闭环拆成明确的角色分工：规划器（planner）负责问题分析、子问题分解、提出迭代信息需求并串联最终作答；时间定位器（temporal grounder）将“抽

象的子问题”转化为检索与执行动作；感知模块根据规划需求在指定时间区间上做可伸缩的采样与理解 [3]。该框架的关键在于把“定位/检索”从主推理链中解耦：离线阶段将视频切分为 clips 并存入多模态嵌入库，在线阶段 grounder 既支持文本查询也支持多模态查询匹配候选片段，并进一步把检索过程组织为“构造—检索—验证—摘要”等动作序列，最后只把经验证的片段作为低层规划结果回传给 planner[3]。在这一闭环里，“验证”不是事后评测，而是定位环节的内生步骤：grounder采用 VLM 充当 verifier 过滤无关片段，借此降低无关上下文引发的幻觉，使 planner 可以把推理预算集中在高层任务分解与证据对齐上 [3]。感知端同样被工具化为“时间粒度可控”的操作：对于细粒度问题，在被定位出的短时间窗内密集采样以提高保真；对于全局性问题，则以更粗的采样覆盖更长区间以保证可扩展性，这形成了以任务为条件的“感知预算调度”接口 [3]。

另一条代表性路线以 Video-CoM 的“Chain of Manipulations (CoM)”为核心，把闭环具体化为多轮交互轨迹 $\tau = \{ ( T _ { 1 } , V _ { 1 } , m _ { 1 } ) , \dots , ( T _ { N } , V _ { N } , m _ { N } ) \}$ ：每一轮在文本推理 $T _ { i }$ 之后选择一个原子操作 $m _ { i }$ ，执行后得到新的视频视图 $V _ { i + 1 }$ 再继续推理，直至输出最终答案 [5]。CoM 的重要贡献是把“工具空间”压缩为少量可组合、可监督的原子操作集合，使闭环的接口足够清晰：模型每步只能在find-segment（按段回看短时间窗）、find-frame（从原视频中隔离出单帧高分辨率证据）、spatial-zoom（裁剪并放大局部区域）三者中选择其一，以主动收集局部化的时空证据 [5]。为了让“定位”动作在模型内可执行，Video-CoM 还将原视频预分段为等长区间，并把 segment/frame 的索引直接叠加在帧上，工具调用等价于在一个离散化的时间—空间坐标系中做可解释的导航与放大 [5]。在此闭环下，工具的角色不仅是提供新像素，更是把“回看、暂停、聚焦”等人类观看动作转译为可学习的离散决策序列，从而使中间步骤可以被标注、被奖励、被消融分析。

VideoTemp-o3 则将闭环抽象为更紧凑的“localize-crop-answer”过程，并强调多轮 refinement：模型支持按需 grounding 与 cropping，并允许在证据不足时多轮细化定位直到获得足够证据再作答[4]。其数据构建进一步把“验证”显式纳入闭环：首先由 Gemini-2.5-Pro 预测候选时间片段并裁剪，然后把裁剪片段与原问题一起输入模型，并明确约束模型只能基于该片段回答；随后进行闭环一致性验证（closed-loop consistency verification），只有当最终答案与 ground-truth 保持一致时样本才被保留，形成多轮、单工具调用训练数据 [4]。对于长视频（ $( > 3 \mathrm { { m i n } } $ ），系统允许额外一轮 refinement才保留为 multi-tool-call 训练数据，从而把“第一次定位失败—再次定位—再裁剪”的真实交互轨迹纳入监督分布 [4]。在强化学习阶段，VideoTemp-o3 还指出仅用 IoU 作为定位奖励会诱发 rewardhacking（模型任意定位以刷分），因此引入带惩罚项的 IoU 奖励以抑制低质量定位并促进更稳定的多轮细化学习，这实质上把“验证/约束”机制做成了可优化的训练接口 [4]。

与上述“闭环即交互推理轨迹”的系统设计相配套，另一类工作更强调工具接口与输出格式的工程化稳定性。VITAL 在训练中通过严格的结构化输出约束与格式奖励来稳定“思考—工具调用—继续思考—作答”的 rollouts：例如将模型输出格式提示为 ...<answer>...</answer>，并对轨迹施加规则化的格式奖励 $R _ { \mathrm { f o r m a t } } ( \tau )$ ，以减少工具调用协议被破坏导致的训练不稳定 [2]。在工具层面，VITAL 的消融结果提示“工具并非越多越好”：在时序定位任务上加入 clip caption 或 clip QA 工具不仅无益，反而会显著拉低 mIoU；相比之下，video clipping（按时间窗裁剪并重新感知）在经验上更能降低幻觉 [2]。这类发现说明闭环中的“感知工具”需要与任务目标和训练信号匹配，否则会把噪声或无关信息引入证据链，反向破坏定位与核验。

在临床长视频场景，MedScope 将闭环组织为交错式视觉链式思维（VCoT）：每轮先输出文本理据，再决定是否调用工具获取证据并将观察结果反馈到下一轮推理，并通过强制的结构化接口<think>（文本理据）、<tool_call>（视觉理据/动作）、<tool_response>（工具观测）、<answer>（终止预测）把推理与取证分离，从而使轨迹既可学习也可审计 [6]。其工具箱刻意保持“轻量但层级化”：crop_video 输入视频与时间窗，返回该窗内密集采样的帧序列用于局部证据搜集；get_frame输入时间戳，返回邻近三帧用于细粒度核验 [6]。也即，“定位/检索”主要依赖时间窗裁剪，而“验

证/复核”更多由近邻帧抽取完成，两者共同实现从粗到细的证据获取闭环，符合临床推理中“先全局扫视再局部回看”的工作流假设 [6]。

如果把上述系统视为“闭环的结构化实现”，那么 Video-Thinker 提供了“闭环的学习化实现”视角：它将工具增强的视频推理形式化为序列决策过程，智能体在生成中间想法与从工具集合中选择动作（如时间 seek、clip extraction、区域 zooming/cropping）之间交替，以获得新的视觉观测，并通过强化学习学习何时、如何调用工具，从而形成由粗到细的证据获取策略 [11]。该框架同时把“效率约束”纳入闭环成本模型：训练奖励除了正确性，还包含对过度工具调用的惩罚，以在性能与交互开销之间做折中 [11]。这使工具接口不再只是可执行操作的集合，而成为可被策略优化的动作空间；闭环是否收敛到“少而关键的证据调用”，取决于奖励塑形能否同时惩罚冗余回看并鼓励命中关键时刻。

值得注意的是，闭环中“定位/检索”并不局限于时间段检索或裁剪。ModaRoute 在多模态视频检索中把“选择检索哪些模态索引”本身工具化为路由决策：LLM 路由器根据查询意图预测信息最可能位于 ASR（语音）、OCR（屏幕文本）或视觉索引之中，只对被选模态并行检索并做排序融合，从而把“何时搜什么”变成可插拔的工具调用前置环节 [7]。其结果显示，相比对三种模态的穷举检索（平均 3.0 模态/查询），路由后平均只需查询 1.78 个模态即可实现约 $4 1 \%$ 的计算开销降低，同时保持 $\mathrm { R e c a l l @ 5 = 6 0 . 9 \% }$ 的检索效果 [7]。当此类路由器被并入 think-with-video 的整体闭环时，它提供了一个更上游的“检索预算分配”接口：在进入视频裁剪/密采样等高成本感知工具前，先用语言理解估计最可能的证据通道，从而改变后续闭环的成本结构与误差模式。

综合这些代表性方法可以看到，典型 think-with-video 的工具调用闭环并非固定模板，而是围绕同一条因果链反复重排接口：规划器或策略负责提出信息需求并决定是否继续取证；定位/检索模块把需求落地为可执行的时间窗、时间戳、片段索引或模态索引选择；感知模块在被约束的证据范围内做密集/高分辨率的重新观测；验证与摘要模块则负责过滤无关检索结果、做闭环一致性检查或生成可供后续推理引用的结构化证据；最终作答模块在证据充分时终止交互并输出结论 [3, 4, 5, 6]。其中，“验证/摘要”既可能以推理时的 verifier/一致性检查出现（如检索结果过滤、闭环一致性验证），也可能以训练时的奖励与数据筛选出现（如格式奖励、证据对齐奖励、对 reward hacking 的惩罚），两者共同决定了闭环能否稳定地产生可追溯的证据链并避免推理漂移 [2, 4, 3]。
