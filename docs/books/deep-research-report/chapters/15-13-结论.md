# 13 结论

本文围绕“think with video”范式中的工具使用（tool using）追问：除局部区域放大（zooming）与拆帧/抽帧等“按帧访问”之外，人类（以及受人类启发的系统设计者）还发明了哪些可复用的视频操作工具与调用机制。综合代表性框架可以看到，工具并不局限于对像素输入做局部变换，而是扩展为一套“按需取证—可验证对齐—再取证”的交互式感知闭环：一方面将时间轴上的检索、定位、裁剪与重看显式化；另一方面将跨模态信息抽取、证据过滤/验证、对象级持续指代与运动表征等能力外部化为可调用组件，并通过结构化轨迹与奖励对齐把这些组件稳定纳入推理过程 [3, 5, 4]。

在“除拆帧以外”的时间维度工具上，最具共性也最直接的增量来自对时间轴的可操作化：不是简单抽取均匀帧序列，而是让模型能够选择“看哪一段、看多长、是否回看、是否细化”。Video-CoM将交互动作显式离散化为 find-segment（按段回看短时间窗）、find-frame（按索引取高分辨率关键帧）与 spatial-zoom，并把推理表述为多轮轨迹 $\tau = \{ ( T _ { i } , V _ { i } , m _ { i } ) \} _ { i = 1 } ^ { N }$ 的循环，强调“回看/重访”作为与拆帧不同的时间操作原语 [5]。VideoTemp-o3 更进一步把任务形式化为 localize–crop–answer：模型先定位候选时间区间，再执行裁剪（cropping）与密集采样（dense sampling）取证，并允许多轮grounding refinement；其数据构建还明确只对长视频（ $\mathrm { { \Omega } ^ { \prime } > 3 \mathrm { { m i n } } \mathrm { { \Omega } } }$ ）启用额外细化回合，以体现“越长越需要迭代式工具调用”的现实约束 [4]。类似地，MedScope 在临床长视频中把“粗到细”的层级检索固化为两个原生工具：crop_video 输入时间窗返回该窗内密集采样的帧序列以收集局部证据，get_frame 输入时间戳返回最邻近的三帧以做细粒度核验，从接口层面把“裁剪—密看—核验”的临床工作流转写为可学习的动作空间 [6]。

与时间定位工具并行演化的是“感知预算调度”这一更接近系统层的工具机制：同样是取帧，关键不在是否拆帧，而在于由推理过程动态决定采样密度与覆盖范围，从而在窗口受限下同时保留全局可扩展性与局部保真度。VideoExplorer 将这一点清晰地制度化：规划器提出子问题后，时间定位器在离线分段嵌入库中检索候选区间并调用验证器过滤，再由视频理解模块按任务选择粗采样或密采样；论文同时以视觉 token 用量对比说明，相比“无差别处理全部片段”的策略，其聚焦高度相关片段的按需访问能够在更少视觉 token 下取得更好性能，因而“采样粒度”本身成为可调度的工具参数而非固定的预处理策略 [3]。

除“去哪看/怎么看”之外，think-with-video 工具谱系的另一条主线是“如何确保看过的东西真的支撑了推理”。这类工具不一定直接改动视频，而是把证据核验与幻觉抑制外显为可调用的验证/评价模块，或把证据对齐转化为可优化的奖励信号。VideoExplorer 的 temporal grounder 在检索后用 VLM 作为 verifier 判定片段相关性，丢弃无关片段并仅返回经验证的证据，从而把“检索—验证—回传”从主推理链中解耦出来以降低漂移与幻觉风险 [3]。VideoTemp-o3 的训练数据流水线包含 closed-loop consistency verification：只有当裁剪片段条件下的答案与 ground-truth 保持一致时样本才被保留，用数据筛选把“片段足以支持答案”的证据充足性变成硬约束 [4]。在训练信号层面，MedScope 以复合奖励 $R = R _ { a c c } + R _ { f o r m a t } + R _ { e v i d e n c e }$ 将证据对齐显式纳入强化学习目标，其中 $R _ { e v i d e n c e }$ 通过时间重叠（对 crop_video）与时间戳对齐（对 get_frame）衡量 grounding 质量，使模型在“答对”之外还必须“用对工具、对齐证据”[6]；VideoTemp-o3 亦指出仅用 IoU 奖励会诱发 reward hacking，并通过 penalty-aware IoU reward 对低质量定位行为施加惩罚以稳定多轮细化学习 [4]。当研究视角从“如何调用工具”转向“如何让推理不漂移”时，工具甚至可以仅在训练期出现：Video-VER 用外部强 MLLM（Qwen2.5-VL-72B）离线生成问题相关的 visual evidence，并以 Visual Evidence Reward（VER）奖励“推理轨迹可验证地扎根于视觉证据”，而策略模型在测试时独立推理、不依赖外部证据生成器；其在 10 个公开基准上 9/10 项排名第一的结果，实质上凸显了“证据对齐评价”作为工具/奖励的价值 [10]。此外，VITAL 的工具消融也提醒：工具并非越多越好，在其时序定位任务上加入 clip caption 或 clip QA 反而会导致 mIoU 显著下降，而 video

clipping（按预测区间裁剪并只在片段上回答）经验上更能降低幻觉，说明工具的收益高度依赖任务设定与工具-模型的协同优化方式 [2]。

从更宽的“工具”定义看，跨模态辅助工具构成了除 zoom/拆帧之外最常被忽略却最具工程决定性的类别：ASR、OCR、字幕/描述生成与跨文档对齐等，把视频中的“可检索符号”提取出来并转化为可索引材料。ModaRoute 将该方向推进为显式的“路由工具”：用 GPT-4.1 预测查询所需模态，在 ASR/OCR/视觉三类索引间做选择性检索，平均每次仅查询 1.78 个模态而非穷举 3.0 个模态，在保持 Recall $\textcircled { \mathrm { a } } 5 { = } 6 0 . 9 \%$ 的同时将计算成本降低约 $4 1 \%$ ；其误差分析显示 OCR 路由准确率仅$4 2 . 9 \%$ ，且存在字幕导致 ASR/OCR 混淆与 OCR-视觉语义重叠等系统性失败模式，直接指向后续tool using 研究中“何时触发 OCR、如何校验 OCR 需求”的关键缺口 [7]。MaViLS 则把“视频—幻灯片对齐”作为另一种结构化索引工具：通过语音转写（faster-whisper/Whisper）、OCR（Tesseract）与图像特征融合，并用动态规划求解最优幻灯片序列；在约 22 小时、12,830 段的视频数据上，其融合方法平均准确率 0.82，相比 SIFT 基线 0.56 且约快 11 倍，并给出了不同模态特征计算的分钟级开销分解（如文本特征平均约 3 分钟）以刻画工具链的真实成本 [8]。面向 OCR 本身的能力瓶颈，VideoOCRBench 进一步将“读文字”“检索文字出现时刻”“基于文字问答”三类任务基准化，并指出在小文本、移动与运动模糊条件下 VLM 性能显著下降、易出现字符幻觉或对短暂文本漏检，提示在 think-with-video 系统中把 OCR 作为工具并不足够，还需要与时间定位、稳像/去模糊、文本区域定位等配套操作共同组成可用的证据获取链路 [9]。

最后，一类常被误归为“模型能力”而非“工具”的机制，是对象级持续指代与运动证据提取。对于需要跨帧追踪同一对象、验证动作连续性或建立因果链的任务，单次拆帧并不能解决“身份一致性”与“跨时刻关联”问题。TAD（Tracking Anything with Decoupled Video Segmentation）将视频目标跟踪解耦为逐帧分割与身份传播：逐帧分割可直接复用强图像分割基础模型生成掩膜，再用mask IoU 与外观嵌入做跨帧关联以维持 identity；这种“可插拔的逐帧感知器 $^ +$ 轻量时序关联器”的工具化结构，为 think-with-video 系统提供了稳定的对象级记忆与指代接口 [21]。Two-shot VOS的双向传播与融合策略则展示了另一种时间工具：在仅有首末帧标注的弱监督条件下，通过前向与后向两路传播并对中间帧融合，使“从两端约束中间过程”的时序推断成为可复用操作序列，尤其适合长视频中目标外观大幅变化或消失再出现的情形 [25]。在运动表征方面，相位（phase）作为光流的替代也可被视作“运动证据工具”：相位变化可表征运动而无需显式估计速度；同时，该工作在实验设置中将“时间导数（如灰度帧时间导数堆叠）”作为输入特征之一，用以在较低开销下捕捉变化并辅助动作识别 [22]。关于“RGB 差分替代光流”“具体加速倍数”等更强的量化结论，现有正文所引来源在公开信息中主要聚焦相位表征本身，因而此处不再给出具体倍率描述，以避免不同工作结论被误并置造成的证据错配风险。

因此，对研究问题的回答是肯定的：除 zooming 与拆帧外，tool using 的主要增量并不来自更多低层视觉变换，而来自把“时间轴可操作化、证据可验证化、跨模态可索引化、对象与运动可持续化”四类机制系统化，并让规划/路由在这些机制之上进行多轮编排 [3, 6, 7]。与此同时，多项工作揭示了一个共同的反直觉观察：工具的引入可能带来性能下降或新的失败模式，尤其当工具输出与任务奖励/评测口径未对齐时（例如工具增加了文本中间层却削弱了时序定位 mIoU，或路由遗漏OCR 导致关键屏幕文本缺失）[2, 7]。这意味着 think-with-video 的核心挑战正在从“能不能调用工具”转向“能否在有限预算下选择正确工具、并用可审计的证据链证明工具调用的必要性与充分性”。

面向后续研究路线，首先需要把工具调用的评价从最终答案扩展为“轨迹级可比性”：一方面沿用并推广结构化接口与格式奖励，使不同系统输出的工具调用（tool call）与观测（observation）可被统一解析与自动评分（如 MedScope 的 <think>/<tool_call>/<tool_response>/<answer> 轨迹接口，以及 VITAL 的严格格式奖励以稳定 rollouts）[6, 2]；另一方面，需要将中间步骤的正确性与证据覆盖纳入指标体系，避免只优化终局准确率导致路径漂移或 reward hacking（Video-CoM

的 step-level reasoning reward、VideoTemp-o3 的惩罚感知 IoU、Video-VER 的证据奖励均指向这一方向）[5, 4, 10]。其次，基准与数据应更系统地覆盖“工具触发条件”而非仅覆盖“工具执行结果”：例如在包含屏幕文字的长视频中，需要同时标注“何时必须 OCR/何时仅视觉足够”的需求标签，并显式纳入对路由错误（漏选 OCR、混淆字幕）的惩罚，从而把 ModaRoute 揭示的失败模式转化为可训练的监督信号 [7]；在长视频时间定位任务中，则应提供可验证的多轮细化轨迹与否定样本（错误定位但答案正确/正确定位但答案错误），以支持对“证据充分性”的更严格评测 [4]。再次，效率指标需要与工具链共同定义：不仅报告准确率或 mIoU，还应同时报告视觉 token、检索/索引成本与端到端运行时（如 ModaRoute 的模态数、MaViLS 的分钟级开销分解、VideoExplorer 的视觉 token 对比），否则不同工具组合之间难以形成可复用的设计结论 [7, 8, 3]。

总之，think-with-video 的工具谱系已经从“看得更清楚”（zoom/拆帧）演进到“看得更对、更省、更可核验”：时间区间检索/裁剪与多轮细化、粗到细的采样调度、证据验证与奖励对齐、跨模态符号抽取与路由、对象级跟踪/分割与运动表征等，构成了除 zooming 与拆帧之外的主要工具类别与机制 [3, 4, 6, 7, 21]。未来更具决定性的突破，可能来自将这些工具在统一接口下组织为可迁移的“证据操作语言”，并用轨迹级数据与指标让不同工具链的收益与代价可重复、可比较、可诊断。
