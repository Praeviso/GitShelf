# 3 相关工作综述与研究脉络

围绕“think with video”的工具使用（tool using）问题，相关研究大致可沿两条主线展开：一条是以 Video-LLM 为代表的“端到端视频编码—语言推理”路线，关注如何在长视频与时序依赖下提升表示能力、训练对齐与评测覆盖；另一条是近年来兴起的 agentic/工具增强范式，将视频理解过程显式外部化为可调用的操作或模块，使模型在推理过程中能够“再观看/再定位/再取证”，从而突破一次性编码带来的语义瓶颈并提升证据可追溯性。

在视频—语言模型与长视频推理方向，早期与主流 Video-LLM 工作通常采用“视频编码器—跨模态投影—LLM”的三段式架构，并通过多阶段训练（大规模视频—文本对预训练、再进行指令微调）将视频理解能力对齐到语言空间，相关综述对这一技术栈、数据集与基准谱系已有系统梳理[14]。例如，Video-LLaMA 将音频与视觉信号与指令微调结合，代表了“端到端对齐到语言空间”的通用基线之一，可作为后文讨论工具增强范式的对照 [15]。在长视频情境下，视频相较图像的挑战集中在时间动态、长程依赖与多模态对齐带来的 token 与计算开销，因而形成了均匀采样、片段级聚合、时序注意力以及引入外部记忆等多种处理路径 [14]。

然而，GLIMPSE 的分析指出，许多“类图像化”的视频基准可以被少量关键帧“扫一眼”式解决，从评测层面削弱了对深时序推理的辨别力；其构建的 3,269 段视频与 $^ \mathrm { 4 , 3 4 2 + }$ 题、11 类任务的视觉中心基准，要求模型观看完整视频并进行跨帧推理，并在人工评测与当前模型之间形成显著差距，揭示了长视频推理在“信息整合与选择”维度仍存在结构性短板 [12]。这类发现进一步说明：单纯增加输入帧数并不能线性提升效果，帧数超过阈值后收益递减甚至下降的现象，提示“如何决

定看什么、何时看、看多细”正在成为比“看多少帧”更核心的瓶颈 [12]。此外，也有工作从提示与启发式角度探索如何让视频—语言基础模型在特定领域的视频问答中更稳健地组织作答步骤与信息使用，这类“提示/策略”可视为后文所讨论“软工具”边界的一个来源 [16]。

在 agentic/工具增强框架方面，Video-CoM 将问题明确归因于“视频被一次性编码后推理完全在文本空间展开”的被动范式，指出其造成模型无法回看、聚焦与验证证据的语义瓶颈，并提出交互式视频推理：把视频当作可操作的认知工作台，通过“Chain of Manipulations”在多轮交互中执行视觉动作以收集并精炼证据 [5]。该框架将工具化操作收敛为三类原子操作：

• find-segment（按段回看短时间窗以重访关键时刻）；  
• find-frame（隔离单帧高分辨率证据）；  
• spatial-zoom（局部裁剪放大以检查细节）。

并将推理过程形式化为包含每步文本与操作的轨迹，从而为“工具调用如何被组织成可解释的多步链”提供了标准化表述 [5]。在训练上，Video-CoM 进一步提出对每一步操作正确性进行度量与奖励的 RA-GRPO，以缓解仅用最终答案监督导致的路径漂移，并在强调操作能力的基准上获得明显提升；其强化学习样本规模仅 3K，表明工具链与逐步奖励可在较小 RL 数据下促成有效的交互策略学习 [5]。

与“工具增强但强调何时调用”的路线相呼应，Video-Thinker 将工具增强视频推理建模为序列决策过程，智能体在“中间思考”与“工具动作选择”之间交替，工具集合涵盖时间 seek、片段trim/extract 以及空间 zoom/crop 等轻量级视频操作，并通过强化学习学习由粗到细的取证策略，同时以工具调用开销惩罚平衡性能与效率 [11]。其消融显示，去除时间维度工具（temporal seeking、clip extraction）会显著降性能，暗示除空间放大之外，“时间导航—片段截取”的工具对长程推理具有不可替代的贡献 [11]。在更强调“时间定位与问答统一”的设计中，VideoTemp-o3 将 thinking-with-videos 形式化为 localize-crop-answer 过程，支持按需裁剪与多轮定位细化；其数据构造采用强模型生成候选时间段并裁剪，再通过闭环一致性验证筛选样本，并对长视频（ $\mathrm { \Omega _ { \mathrm { \ell } } > 3 m i n }$ ）进一步启用多轮 refinement 生成多工具调用轨迹，体现了工具调用次数与视频时长/任务难度的耦合关系 [4]。这些工作共同推动了一个研究脉络：工具不再仅是“更强的视觉编码器”，而是推理链中的可执行操作与决策对象，学习目标从“答对”扩展为“以怎样的交互轨迹答对”。

在视频检索与时间定位研究线中，VideoExplorer 更系统地把“规划—时间定位—可伸缩感知”整合为统一范式：规划器分解复杂任务并提出子问题，时间定位器在离线分段的多模态嵌入库上进行在线匹配，并把检索过程与主推理链解耦，支持构造、检索、验证、总结等动作并将经验证片段返回规划器 [3]。其进一步强调不同任务对应不同时间粒度需求：对细粒度问题在定位到的时间窗口内进行密采样，对全局问题覆盖更大时间区间并采用粗采样，以在上下文窗口受限下平衡可扩展性与细节保真 [3]。该框架以“更少视觉 token 但更好性能”的证据说明按需检索与预算调度的有效性，并指出相较若干检索增强 agent 框架，其时间定位精度更高，核心原因在于分步推理与片段证据的对齐 [3]。与之相对，VITAL 这类工具增强框架也从消融角度强调“工具选择并非越多越好”：在时序定位任务中，加入 clip caption 或 clip QA 工具不仅无益甚至会造成 mIoU 显著下降，而 videoclipping 在经验上更能降低幻觉，提示工具与任务/评测设定存在强耦合，需要系统化的工具有效性分析与组合策略 [2]。

围绕视频采样与效率优化，除传统均匀抽帧外，越来越多工作把“关键时刻/片段选择”视为可独立模块甚至外部工具：Moment Sampling 以通用文本到视频的 moment retrieval 模型指导帧采样，提出与下游 VideoLLM 解耦、无需再训练的 query-aware 选帧方式，先用 QD-DETR 预测相关时间片段并将相关性分数转为帧级得分，再结合高斯平滑、质量分（以拉普拉斯方差检测模糊）、均

匀性分散约束与特征聚类去冗余，最终用贪心策略选帧 [13]。这种“检索信号 采样决策”的工具链与 GLIMPSE 所揭示的“帧数增加但收益下降”现象形成呼应：长视频推理的关键不在于输入更多帧，而在于用更强的路由/筛选机制把感知预算聚焦到与问题相关的证据上 [12, 13]。

与采样效率互补的是“检索什么模态”的路由问题。ModaRoute 将多模态视频检索拆解为对ASR、OCR 与视觉索引的选择性查询，并用 LLM 对查询意图进行分析以动态选择最相关模态，再对被选模态并行检索并融合排序；其在评测中以平均每次仅查询 1.78 个模态（对比穷举 3.0 模态）实现计算开销显著降低且保持较高召回，揭示了工具增强不仅发生在视频剪辑/裁剪层，也可发生在检索层的“信息源路由”层 [7]。该工作同时指出 dense captions 尽管可获得更高召回，但离线处理代价高且对含场景文本而 ASR 未覆盖的片段存在遗漏，为后续将 OCR/字幕等作为显式工具并与路由策略耦合提供了量化动机 [7]。

在“证据可靠性与推理漂移”相关研究中，When Thinking Drifts 系统分析了将 CoT 直接用于视频推理可能导致性能下降与“visual thinking drift”，并提出 Visual Evidence Reward（VER）在强化学习中显式奖励“可验证地扎根于视觉证据”的推理轨迹；其训练阶段借助强 MLLM 生成问题相关视觉证据，但推理阶段不依赖外部模型，从而将工具（教师/证据生成器）限定在训练期使用以降低测试期开销与反馈环风险 [10]。该工作同时指出失败模式之一来自不完整帧采样导致关键信息缺失，进一步说明“采样/定位工具”与“证据对齐/验证工具”在长视频推理中是相互制约的共同体 [10]。

最后，视频生成式推理范式为 think-with-video 的研究脉络提供了另一种路径：与“通过工具取证后作答”的范式不同，视频生成式推理主张将推理过程外显为可生成、可检查的中间视频，从而把“想象/演算”置于视频表征空间中，以期获得更强的多模态推理与可验证性 [1, 17]。在这一谱系里，工具的含义可能从“检索与裁剪”等外部操作扩展为“生成—对齐—对比”的内部演算机制，并与现有以检索/定位为中心的工具链形成互补：前者更像在视频空间中构造可检验的中间变量，后者更像在真实视频证据上做按需的取样与验证。

综合来看，现有综述多以“模型架构—训练数据—任务基准”的视角组织 Video-LLM 研究版图[14]。而近期 think-with-video 相关工作则把注意力转向推理过程本身的可操作性：通过规划、时间定位、裁剪、密/粗采样切换、模态路由与证据验证等模块，将“看视频”重构为可学习的工具调用序列 [3, 4, 5, 11, 13, 7, 10]。在此背景下，引入“工具谱系（tool lineage）”视角具有必要性：以人类观看视频时可执行的操作为原型，对齐到现有系统中的可调用工具、接口形态与训练对齐方法，并分析工具之间的组合关系（如检索—验证—再检索、由粗到细、预算调度）及其在不同任务与评测设定下的增量价值，从而更具体地解释为何某些工具有效、某些工具在特定设定下反而有害 [2, 3, 12, 13, 7]。
