# 6 工具类别一：时间维度的检索、定位与裁剪工具

在“think with video”范式中，围绕时间轴的工具之所以被频繁外部化（或被显式建模为可调用模块），根本原因在于长视频的核心瓶颈往往不是“是否看见某一帧”，而是“在何时、以何种粒度、以多大预算去看哪些时间片段”，以及“如何把时间段证据可靠地回传给推理链”。与“拆帧/抽帧”相比，时间维度工具把视频访问从一次性的静态采样，升级为可迭代的检索—裁剪—复核闭环：模型能够先在全局建立假设，再提出子问题并定位候选时间窗，随后对该窗进行密集观察或重新观看，必要时继续细化时间边界直至获得足够证据 [3, 4]。

时间区间定位（temporal grounding）的一个关键增量价值，是将“抽象的信息需求”转译为“可执行的时间窗动作”。VideoExplorer 将该过程形式化为以子问题驱动的迭代机制：给定问题 $\mathbf { q }$ ，规划器在第 $t$ 轮产生子查询 $\mathbf { q } _ { t } ^ { \mathrm { s u b } }$ ，时间定位器根据子查询输出时间跨度 $\tau _ { t } = \Gamma _ { \mathrm { g r o u n d } } ( \mathbf { q } _ { t } ^ { \mathrm { s u b } } )$ ，感知模块再对该时间跨度执行可伸缩的理解 $\mathcal { P } _ { t } = \Phi _ { \mathrm { p e r c } } ( \tau _ { t } )$ ，最终得到答案 $\mathcal { V } = \Theta ( \mathbf { q } \mid \{ \mathcal { P } _ { t } \} _ { t = 1 } ^ { T } ) |$ [3]。这种“先定位、后感知”的分工，使得时间定位不仅是检索返回一个片段，更是推理链中对证据边界与预算的显式控制点：当子问题需要细粒度动作/瞬时事件时，系统可以在定位到的窄时间窗内密集采样以

提升保真；当子任务是全局角色、主题等宏观问题时，则覆盖更宽区间并采取粗采样以提升可扩展性 [3]。

“片段检索/重访（rewatch/find-segment）”进一步把时间定位落到可复用的原子操作上。Video-CoM 明确将推理建模为多轮交互轨迹，并在每一步从三种原子操作中选择其一：find-segment、find-frame 与 spatial-zoom；其中 find-segment 的定义是从原视频按段索引检索短时间片段$V _ { i + 1 } = g _ { \mathrm { c l i p } } ( V _ { 0 } , s )$ ，用于回看快速动作或关键瞬间，find-frame 则按帧索引抽取单帧高分辨率画面以冻结证据 [5]。为降低时间定位的操作成本与歧义，Video-CoM 还把视频预先切分为等长区间，并将 segment/frame 索引直接叠加在画面上，让“定位到哪一段/哪一帧”成为可验证的离散决策 [5]。这一设计的意义在于：即便同样是“抽帧”，当抽帧被组织为“在时间轴上跳转并重看”的可追溯动作时，它对推理过程提供的是可回溯的证据链，而非一次性输入的一组静态帧。

视频裁剪/截取（video clipping/cropping）是时间工具中最接近“人类播放器操作”的一类：先圈定时间窗，再把该窗作为后续推理的唯一证据来源，以减少无关片段带来的幻觉与漂移。VideoTemp-o3 将 thinking-with-videos 直接刻画为“localize-crop-answer”过程，并强调系统支持按需裁剪（on-demand grounding and cropping）以及多轮定位细化：模型可以在获得初始候选时间段后裁剪该段并据此作答，若证据不足则再进行一轮（或多轮）重定位与再裁剪，迭代直至收集到足够可靠的证据 [4]。其数据构建流程也把裁剪当作强约束：首先由外部强模型预测候选时间段并裁剪，将裁剪片段与原问题输入模型，同时“显式约束模型只能基于该片段回答”；随后通过闭环一致性验证，仅保留答案与标注一致的样本，并在长视频（ $\mathrm { \hbar > 3 m i n } .$ ）上进一步允许额外一轮 refinement，形成多工具调用的多轮训练数据 [4]。这种把“裁剪后作答”写入数据与训练约束的做法，使裁剪不再只是工程加速手段，而成为提高证据可追责性的推理协议。

在临床长视频等高风险场景，裁剪往往被组织成“粗到细（coarse-to-fine）”的层级检查工具链。MedScope 提出交错式视觉链式思维（VCoT），在文本假设与工具取证之间交替推进，并给出了两种原生工具来支持层级化时间检查：crop_video 接受时间窗并返回该窗内“片段级的密集采样帧序列（clip-level densely sampled frame sequence）”用于局部取证；get_frame 接受时间戳并返回最邻近的三帧用于精细核验 [6]。这类工具的增量价值在于把“先粗检再精查”的临床观看习惯制度化：模型可以先用较粗时间窗定位候选操作步骤或关键转折，再用时间戳邻域的少量帧验证细节，从而在长程序视频中实现可控的证据寻访与复核循环 [6]。

时间分段索引与多模态嵌入库，使“定位/检索”可以在离线预处理与在线推理之间重新分配成本。VideoExplorer 提出在离线阶段把视频分割为 clips 并存储多模态嵌入；在线阶段由时间定位代理对查询与候选片段进行匹配，并将检索动作从主推理链中解耦出来。该代理不仅执行检索，还会把子查询转化为构造、检索、验证、总结等具体动作，并将经验证的片段返回给规划器 [3]。关键点不在于“是否有一个向量库”，而在于把时间定位实现为可执行子系统：它可接受文本或多模态查询，借助验证器过滤无关片段，并以较低认知负担向主推理链提供“已定位且已校验”的时间证据，从而降低无关片段引发的幻觉风险 [3]。

粗到细的多轮定位细化（multi-turn refinement）通常与“证据是否充足”的自我评估绑定：先产出一个低成本的粗定位，再根据回答失败或证据不充分的信号触发进一步收缩时间边界。VideoTemp-o3 将这种细化机制明确为多轮 grounding refinement，并在数据侧区分单工具调用与多工具调用轨迹，以覆盖“第一次裁剪就足够”与“需要再次裁剪”两类行为分布 [4]。MedScope 则从工具层面给出层级化动作空间，并在强化学习奖励里把“最终一次工具调用的证据对齐”显式纳入优化目标：证据项根据 crop_video 的时间窗重叠（temporal overlap/IoU）与 get_frame 的时间戳对齐进行评估，并对 crop_video 额外采用 IoU 奖励日程（bonus schedule）以鼓励更精确的时间窗选择 [6]。这种把“时间对齐质量”写进奖励的做法，实质上把多轮细化从启发式策略提升为可学习的行为：模型不仅要答对，还要在时间轴上把证据找得对。

在时间维度工具链中，“视频裁剪”相对“拆帧”的另一个增量价值是对幻觉的经验性抑制：当模型被迫围绕一个明确时间窗重看/密集观察时，它更容易在局部证据上达成一致，而不是在全局稀疏帧中用语言先验填空。VITAL 在对不同视觉工具的消融中观察到，加入 clip caption 或 clip QA工具不仅未提升 temporal grounding 的结果，甚至会导致 mIoU 显著下降；相反，video clipping在经验上相较其他工具更能减少幻觉 [2]。这提示一个重要工程事实：时间工具的有效性不仅取决于“信息量”，还取决于工具输出是否会引入新的语义噪声（例如自动字幕/问答的偏差），以及工具是否能把证据边界收紧到足以支撑可验证判断的时间窗。

与“定位片段后再密采样”并行的一条路线，是先用“时刻/片段检索模型”指导采样，从而在不显式多轮交互的情况下提高时间证据命中率。Moment Sampling 工作提出利用通用文本到视频moment retrieval 模型来引导长视频问答的帧选择：其核心是使用轻量 moment retrieval（采用在QVHighlights 上训练的 QD-DETR）从“视频 $^ +$ 问题”预测相关 moments，并将 moment 相关性转为帧级得分；由于 moments 边界较硬，作者对帧级相关性进行高斯平滑以获得时间连续性，并叠加质量分（以拉普拉斯方差检测运动模糊）、均匀性分（鼓励时间分散）与聚类抑冗余等项，最终用贪心策略选取帧集合 [13]。从工具视角看，这相当于把“时间定位”封装为一个可插拔的检索器，为下游 VideoLLM 提供 query-aware 的时间优先级信号，弥补均匀采样在长时程推理中容易遗漏关键帧的缺陷 [13]。

最后，“何时检索什么”的路由策略并不只发生在时间轴内部（选哪一段），也发生在跨索引/跨工具的选择层面（先走哪条证据通道、花多少成本）。ModaRoute 把这一问题明确为由 LLM 驱动的检索路由：系统用 GPT-4.1 分析自然语言查询的结构与语义，预测应检索 ASR、OCR 与视觉索引中的哪些模态，并只查询被选模态，随后并行检索并通过排序融合得到最终结果 [7]。在其实验中，路由策略将平均每次查询的模态数从穷举的 3.0 降到 1.78，在 R $\mathrm { e c a l l @ 5 = 6 0 . 9 \% }$ 的同时实现约 $4 1 \%$ 的计算开销降低 [7]。对 think-with-video 系统而言，这类“路由工具”提供了一种更上层的时间预算调度方式：当问题的关键信息更可能存在于语音/屏幕文本/纯视觉事件的某一通道时，先做模态选择可以显著减少无效检索，再把剩余预算投入到真正需要的时间窗裁剪与密集观察中。

综上，时间维度工具把视频推理从“对一堆帧做一次性理解”重构为“在时间轴上可操作的证据搜寻过程”：时间区间定位提供可执行的证据边界，find-segment/rewatch 将回看固化为原子动作，裁剪与密采样把证据收缩到可验证窗口，分段嵌入库与解耦的 grounder 把检索从推理链中外部化并可插入验证/摘要，多轮细化与奖励对齐把“找得更准”变成可学习目标，而路由策略则决定在资源受限时优先打开哪条检索通道 [2, 3, 4, 5, 6, 13, 7]。
